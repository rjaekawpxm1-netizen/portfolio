AI 스트레스 분석


1. 프로젝트 개요

본 프로젝트는 사용자 데이터 기반 스트레스 상태를 예측하기 위해
머신러닝 모델을 활용한 AI 스트레스 분석 시스템을 구현한 프로젝트입니다.

데이터 분석에 그치지 않고,
분석 결과가 실제 서비스(앱)로 확장될 수 있는 구조를 염두에 두고 설계했습니다.

2. 문제 정의 (Problem Statement)

스트레스는 개인의 컨디션과 생산성에 큰 영향을 미치지만, 다음과 같은 한계가 존재합니다.

스트레스 상태를 객관적으로 판단하기 어려움

설문 또는 주관적 응답에 의존

지속적인 관리 및 추적이 어려움

👉 본 프로젝트에서는
“사용자 데이터를 통해 스트레스 상태를 정량적으로 예측할 수 있을까?”
를 핵심 문제로 설정했습니다.

3. 데이터 설명

사용자 상태 관련 데이터

주요 변수 예시:

생활 패턴 데이터

신체/행동 관련 지표

스트레스 상태 라벨(또는 점수)

※ 실제 서비스 환경에서도 확장 가능하도록
일반화된 변수 구조를 가정하여 설계했습니다.

4. 분석 및 모델링 접근 방식
4-1. 데이터 전처리

결측치 및 이상치 처리

분석 목적에 맞는 피처 정제

모델 학습을 위한 데이터 구조화

4-2. 스트레스 예측 모델링

머신러닝 모델을 활용한 스트레스 상태 분류

입력 변수와 스트레스 수준 간 관계 학습

모델 성능 평가를 통한 안정성 확인

5. 핵심 분석 내용
5-1. 주요 변수 영향 분석

스트레스 예측에 영향을 미치는 핵심 변수 확인

단일 지표가 아닌 복합적인 패턴이 스트레스에 영향을 미침을 확인

5-2. 예측 결과 해석

스트레스 수준을 단순 점수가 아닌 **상태(Level)**로 분류

사용자에게 직관적으로 전달 가능한 형태로 결과 설계

6. 문제 발생 및 해결 과정
문제 1. 스트레스라는 개념의 모호함

문제 상황
스트레스는 정답이 명확하지 않은 추상적인 개념으로,
모델 학습 기준을 설정하기 어려웠습니다.

해결 과정

스트레스를 연속값이 아닌 **범주형 상태(Level)**로 정의

예측 결과의 해석 가능성에 초점

해결 결과

모델 결과를 사용자에게 설명 가능한 형태로 제공 가능

문제 2. 모델 성능보다 해석 가능성의 중요성

문제 상황
단순히 정확도가 높은 모델은
실제 서비스 적용 시 신뢰를 얻기 어렵다고 판단했습니다.

해결 과정

성능과 함께 변수 영향 해석에 집중

“왜 이런 결과가 나왔는지” 설명 가능한 구조 설계

해결 결과

AI 결과에 대한 사용자 신뢰도 향상 가능성 확보

문제 3. 분석 결과의 활용 시나리오 부족

문제 상황
분석 결과가 모델 출력값에서 끝날 위험이 있었습니다.

해결 과정

스트레스 분석 결과를

앱 화면

상태 피드백

추후 관리 기능으로 확장 가능한 구조로 설계

해결 결과

분석 → AI → 서비스로 이어지는 흐름 완성
