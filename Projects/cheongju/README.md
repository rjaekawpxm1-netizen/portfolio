청주 교통사고 데이터 분석


1. 프로젝트 개요

본 프로젝트는 청주시 교통사고 공공데이터를 활용하여
사고 발생 패턴과 위험 요인을 분석하고,
지역 교통 안전 개선에 활용 가능한 인사이트를 도출하는 것을 목표로 합니다.

정량적 데이터 분석뿐 아니라,
분석 결과를 자연어 기반으로 질의·요약할 수 있는 LLM 환경까지 확장한 것이 특징입니다.

2. 문제 정의 (Problem Statement)

교통사고 데이터는 다음과 같은 한계를 가집니다.

사고 건수는 많지만 어디가, 언제, 왜 위험한지 직관적으로 파악하기 어려움

일반 사용자나 비전문가는 분석 결과를 직접 해석하기 어려움

데이터 분석 결과가 보고서 수준에서 끝나는 경우가 많음

👉 본 프로젝트에서는
“데이터 분석 결과를 어떻게 더 쉽게 이해하고 활용할 수 있을까?” 를 핵심 문제로 설정했습니다.

3. 데이터 설명

청주시 교통사고 공공데이터

주요 변수 예시:

사고 발생 위치

발생 시간대

사고 유형

피해 규모

도로 및 환경 조건

본 프로젝트는 역할이 다른 두 개의 폴더로 구성되어 있습니다.

5. analysis 폴더 설명 (데이터 분석 영역)
역할

교통사고 원본 데이터를 기반으로 한 정량적 데이터 분석

사고 패턴, 위험 요인, 시간·공간적 특성 파악

주요 내용

데이터 전처리 및 정합성 확인

사고 유형별 / 시간대별 / 지역별 분석

시각화를 통한 위험 구간 및 패턴 도출

목적

👉 “어디에서, 언제, 어떤 사고가 많이 발생하는가?” 를 데이터로 설명

6. llm_rag 폴더 설명 (분석 결과 활용 영역)
역할

analysis에서 도출된 결과를 기반으로

자연어 질문에 답변할 수 있는 LLM 환경 구성

주요 내용

분석 결과 요약 데이터 생성

벡터화 및 검색(Retrieval)

LLM을 활용한 질의응답 구조 구현

예시 질문

“청주에서 교통사고가 가장 많이 발생하는 시간대는?”

“사고 위험이 높은 지역의 특징은?”

“야간 사고의 주요 원인은 무엇인가?”

목적

👉 분석 결과를 비전문가도 쉽게 활용 가능하도록 확장

7. 분석 접근 방식
7-1. 데이터 분석 (analysis)

EDA를 통해 사고 분포 및 특성 파악

단순 건수 비교가 아닌 패턴 중심 분석

지역·시간·사고 유형 간 관계 분석

7-2. 분석 결과 활용 (llm_rag)

분석 결과를 문서 형태로 구조화

RAG 방식을 통해 LLM이 필요한 정보만 검색 후 응답

데이터 기반 설명이 가능한 질의응답 환경 구성

8. 주요 인사이트

특정 시간대와 지역에서 사고가 집중되는 패턴 존재

사고 유형에 따라 위험 요인이 명확히 구분됨

데이터 분석 결과를 자연어로 제공할 경우 활용성과 접근성이 크게 향상됨

9. 문제 발생 및 해결 과정
문제 1. 분석 결과 해석의 진입 장벽

비전문가가 결과를 이해하고 활용하기 어려웠습니다.

해결 과정

분석 결과를 요약 문서로 정리

LLM + RAG 구조를 도입하여 질의응답 형태로 제공

해결 결과

비전문가도 질문을 통해 인사이트 획득 가능
