📊 고객 이탈 분석 (Churn Analysis Dashboard) – Tableau 프로젝트

이 프로젝트는 고객의 이탈(Churn) 여부에 따라

평균 주문 건수

평균 구매 주기

이탈/유지 비율
등을 직관적으로 분석하기 위해 Tableau로 대시보드를 구축한 프로젝트입니다.

데이터는 옵션 주문 기록 기반으로 구성되었고,
Churn Label을 기준으로 고객을 이탈(1) vs 유지(0) 로 나누어 분석했습니다.

🗂 프로젝트 구성
1. 데이터 전처리 (Python)

Tableau로 가져오기 전 Python에서 다음 계산 필드를 준비했습니다:

Days Between
→ 첫 구매일과 마지막 구매일의 차이

Avg Days Per Order
→ Days Between / Total Orders

⚠ 중요: 평균값이 생각보다 매우 작거나(1~3 수준), 반대로 매우 크게(최소 0일~최대 170일 이상) 분포하여
Tableau의 축 자동 설정이 극단적으로 잡히는 문제가 발생했습니다.

즉, 차트가 안 보이던 이유는 데이터가 없는 게 아니라, 축 범위 때문에 가려졌던 것입니다.

→ 해결: 축 범위를 직접 조절(0~10, 0~200 등) 하여 문제 해결.

📌 최종 Tableau 대시보드 구성
✔ 시트 1 – 이탈/유지 비율 파이 차트

고객 분포를 한눈에 확인하는 시각화입니다.
이탈 고객의 비율이 어떻게 구성되는지 쉽게 파악할 수 있습니다.

🔍 결과 이미지

✔ 시트 2 – 평균 주문 건수 & 평균 구매 주기 비교

Churn Label(0=유지, 1=이탈) 별로

평균 주문 건수 (Avg Total Orders)

평균 구매 주기 (Avg Days Between)
를 비교하여, 이탈 고객의 행동적 특징을 분석합니다.

🔹 평균 주문 건수 비교

유지 고객이 이탈 고객보다 평균 주문 건수가 높음
→ 충성도 높은 고객일수록 이탈하지 않음


🔹 평균 구매 주기 비교

유지 고객이 더 짧은 주기로 재구매함
→ 구매 간격이 긴 고객일수록 이탈 가능성이 높음


✔ 시트 3 – 산점도 분석 (Days Between vs Avg Days Per Order)

두 지표가 선형적으로 증가하는 형태를 띄며,
비슷한 패턴을 보이는 고객군들이 뚜렷하게 나타납니다.

→ 엘보우처럼 보이는 선형 패턴은 고객의 주문 패턴 특성을 보여줌.

🧠 분석 요약
🔸 유지 고객(Churn=0)은…

평균 주문 건수가 높다

구매 주기가 짧다 → 재구매가 빠른 고객

이탈 가능성이 낮다

🔸 이탈 고객(Churn=1)은…

평균 주문 건수가 현저히 적다

구매 주기가 매우 길다

다시 구매하지 않거나 간헐적으로 구매했을 가능성이 크다

🔸 시각화가 제대로 표시되지 않았던 이유

Avg Days Per Order 값이 평균 1~3 수준으로 매우 낮았음

Tableau가 축 범위를 자동으로 잡으면서
모든 값이 하단에 붙어버려서 보이지 않음 → 차트가 없는 줄 알았던 상황

축을 수동으로 설정하여 문제 해결

📁 최종 구성된 Dashboard 예시

전체 3개 시각화를 하나의 대시보드에 배치하여
이탈 분석을 종합적으로 확인할 수 있도록 구성했습니다.

🚀 배운 점 & 문제 해결 경험
🔧 1. 평균값이 작아서 그래프가 보이지 않던 문제 해결

평균이 너무 작은 값(0~3)

Tableau가 자동 축 범위를 크게 잡아버려서
→ 그래프가 존재해도 눈에 보이지 않는 문제 발생

해결 방법
✔ 축 범위를 수동으로 0~10 또는 0~200 등으로 설정
✔ 차트 형태를 막대/선/산점도로 변경하면서 최적의 표현 방식 선택

💡 2. 계산 필드 생성과 시각화 매칭 오류 해결

Avg Days Per Order 같은 필드를 Sum으로 가져오면 왜곡된 값이 나오는 문제

해결
✔ 집계 방식을 Average로 직접 변경
✔ 시트마다 집계 방식이 달라지지 않도록 고정 설정

Python (Pandas) – 데이터 전처리, 가공

Tableau Desktop – 데이터 시각화

Markdown – 분석 문서화

Tableau 시각화에서 발생하는 축 설정 문제를 해결하는 중요한 경험도 얻었습니다.
