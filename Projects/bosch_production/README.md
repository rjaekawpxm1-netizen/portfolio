제조 공정 불량 분석 및 예측 모델 구축

캐글데이터(Bosch Production Line Performance)

대규모 제조 공정 데이터를 분석하여 불량 발생이 집중되는 공정 구간을 도출하고, 공정 흐름 패턴과 불량 간의 관계를 확인한 프로젝트입니다.
특정 공정 라인에서 평균 대비 높은 불량률이 발생함을 확인하고, 이를 기반으로 개선 가능성을 제시했습니다.
또한 공정 흐름 정보를 활용하여 불량 제품을 사전에 예측하는 모델을 구축했습니다.

1. 프로젝트 개요

본 프로젝트는 Bosch 생산 라인 데이터를 기반으로
제품이 생산 공정을 통과하는 과정에서 발생하는 불량 패턴을 분석하고,
불량 발생이 집중되는 공정 구간을 탐색하여 개선 가능성을 도출하는 것을 목표로 합니다.

단순히 불량 여부를 예측하는 것을 넘어,
불량이 발생하는 공정 구조를 이해하고 실제 운영 개선에 활용 가능한 인사이트를 도출하는 데 초점을 두었습니다.

2. 문제 정의 (Problem Statement)

대규모 제조 공정에서는 다음과 같은 문제가 반복적으로 발생합니다.

불량 제품이 어떤 공정 단계에서 발생하는지 명확히 파악하기 어려움

공정 데이터는 방대하지만, 실제 문제 구간을 특정하기 어려움

불량 발생을 사전에 예측하거나 예방할 기준 부족

따라서 본 프로젝트에서는 다음 질문을 핵심 문제로 설정했습니다.

👉
“불량 제품은 어떤 공정 흐름에서 발생하며, 문제 공정 구간을 찾을 수 있는가?”

3. 데이터 설명

Kaggle Bosch Production Line Performance 데이터셋을 사용하였습니다.

데이터에는 다음과 같은 정보가 포함됩니다.

제품별 공정 통과 기록

각 공정에서 측정된 시간 정보

공정 단계별 측정 존재 여부

최종 불량 여부(Response)

데이터는 수천 개의 공정 측정 변수로 구성되어 있으며,
실제 제조 공정의 복잡한 흐름을 반영합니다.

4. 분석 접근 방식
4-1. 기본 분석

초기 단계에서 다음을 확인했습니다.

불량 제품 비율 확인 (심각한 클래스 불균형 존재)

공정 소요 시간 분포 분석

공정 통과 패턴 및 결측 패턴 분석

4-2. 핵심 가설

“불량 제품은 정상 제품과 다른 공정 흐름을 가질 것이다.”

이를 검증하기 위해 다음 관점에서 분석을 진행했습니다.

공정 소요 시간과 불량의 관계

공정 통과 횟수 차이 분석

측정 누락 패턴 분석

특정 공정 라인에서의 불량 집중 여부 분석

5. 핵심 분석 내용
5-1. 공정 시간 패턴 분석

분석 결과,

공정 시간이 극단적으로 짧거나 긴 제품에서 불량률 증가

정상 제품은 비교적 안정적인 공정 흐름을 유지

즉,

📌
공정 흐름이 정상 패턴에서 벗어날수록 불량 발생 가능성이 증가함을 확인

5-2. 문제 공정 라인 탐색

각 공정 station별 불량률을 분석한 결과,

특정 라인 구간에서 불량이 집중되는 현상이 발견되었습니다.

특히,

L3 라인의 S32 공정


에서 평균 대비 약 10배 높은 불량률이 관찰되었습니다.

이는 해당 공정 또는 해당 구간 공정 흐름에 문제가 존재할 가능성을 시사합니다.

5-3. 문제 공정 이전 흐름 분석

문제 공정 이전 단계에서도 동일한 공정 흐름을 거치는 제품이 많음을 확인했으며,

이는 단일 공정 문제가 아닌 공정 구간 전체의 영향 가능성을 보여줍니다.

6. 주요 인사이트

불량 제품은 정상 공정 흐름에서 벗어나는 경향 존재

공정 시간 패턴이 품질과 밀접하게 연관됨

특정 공정 라인 구간에서 불량이 집중됨

문제는 단일 공정이 아닌 공정 흐름 전체 영향 가능성 존재

7. 개선 시나리오 제안

분석 결과를 기반으로 다음 개선 방향을 제안할 수 있습니다.

문제 공정 구간 설비 상태 점검

해당 공정 진입 전 추가 품질 검사 수행

공정 흐름 이상 실시간 감지 시스템 구축

공정 조건 변화에 대한 지속적 모니터링 체계 도입

9. 프로젝트 의의

본 프로젝트는 데이터 분석을 통해
실제 제조 공정 운영 개선에 활용 가능한 인사이트를 도출한 사례입니다.

단순 예측 모델 구축을 넘어,

공정 흐름 구조를 이해하고

문제 구간을 도출하며

개선 방향까지 연결했다는 점에 의의가 있습니다.

10. 문제 발생 및 해결 과정
문제 1. 공정 데이터 규모 및 복잡성 문제

문제 상황
데이터 규모가 매우 크고 공정 변수 수가 많아 분석 방향 설정이 어려웠습니다.

해결 과정

전체 데이터 대신 샘플 기반 분석 진행

공정 흐름 요약 feature 생성

단계별 분석 구조 수립

해결 결과

공정 흐름을 단순화하여 분석 가능 구조 확보

문제 2. 불량 원인 특정의 어려움

문제 상황
초기에는 전체 공정 평균만으로는 문제 구간을 파악하기 어려웠습니다.

해결 과정

station 단위 불량률 분석 수행

공정 라인 단위 비교 진행

해결 결과

불량이 특정 공정 구간에 집중됨을 확인

문제 3. 분석 결과와 모델링 연결 문제

문제 상황
분석 인사이트를 예측 모델에 어떻게 반영할지 명확하지 않았습니다.

해결 과정

문제 공정 방문 여부를 feature로 추가

공정 흐름 기반 feature 생성

해결 결과

분석 결과가 실제 모델 예측에 활용됨을 확인

11. 분석을 통해 얻은 교훈

제조 데이터 분석에서는 공정 흐름 이해가 중요하다

단일 변수 분석보다 공정 경로 분석이 더 중요한 인사이트를 제공한다

데이터 분석은 단순 설명이 아니라 의사결정 지원까지 연결되어야 한다
