호텔 예약 위험도 분석 (Hotel Booking Risk Analysis)
1. 프로젝트 개요

본 프로젝트는 호텔 예약 데이터를 기반으로
예약 조건에 따라 발생하는 취소 및 매출 손실 위험 요인을 분석하고,
비즈니스 관점에서 관리 가능한 고위험 예약 조합을 도출하는 것을 목표로 합니다.

단순 취소율 분석이 아닌,
실제 손실 규모와 연결된 위험 구조를 파악하는 데 초점을 두었습니다.

2. 문제 정의 (Problem Statement)

호텔 운영 과정에서 다음과 같은 문제가 반복적으로 발생합니다.

취소율은 높은데, 실제 손실로 이어지는 예약 유형은 무엇인지 불분명

환불 불가 / 보증금 / 예약 채널 조합에 따른 위험 관리 기준 부재

취소가 발생해도 “얼마나 위험한 취소인지” 판단하기 어려움

👉 따라서 본 프로젝트에서는
“어떤 예약이 취소될 때 가장 큰 손실을 만드는가?” 를 핵심 질문으로 설정했습니다.

3. 데이터 설명

호텔 예약 데이터 (기간별 예약 및 취소 기록)

주요 변수 예시:

예약 채널 (Direct, OTA, Travel Agency 등)

환불 여부 (Refundable / Non-refundable)

보증금 유형

숙박 요금

예약 상태 (취소 여부)

4. 분석 접근 방식
4-1. 기본 분석

예약 유형별 취소율 분석

환불 조건, 채널별 분포 확인

단순 취소율과 손실 규모의 차이 비교

4-2. 핵심 가설

“취소율이 높은 예약이 반드시 가장 위험한 예약은 아닐 수 있다.”

이를 검증하기 위해 취소 발생 시 손실 금액을 중심으로 분석을 진행했습니다.

5. 핵심 분석 내용
5-1. 예약 조건 조합별 손실 분석

환불 여부 × 예약 채널 × 보증금 조합 생성

조합별:

취소 건수

총 손실 금액

평균 손실 규모 분석

5-2. 고위험 예약 조합 정의

취소율은 상대적으로 낮지만,

취소 시 손실 규모가 큰 예약 유형을 고위험으로 분류

📌 결과적으로

“환불 불가 예약 자체가 위험한 것이 아니라,
특정 유통 채널과 결합될 때 관리가 어려운 위험으로 전환된다”는 인사이트를 도출했습니다.

6. 주요 인사이트

단순 취소율 기준만으로는 실질적인 위험 판단이 어렵다

손실 규모 기준 분석을 통해 관리 우선 대상 예약 유형 식별 가능

일부 예약 조건은 정책 조정(보증금, 채널 조건)만으로도 위험 완화 가능

7. 개선 시나리오 제안

분석 결과를 바탕으로 다음과 같은 개선 방향을 제안했습니다.

고위험 조합에 대한:

보증금 조건 조정

예약 채널 정책 개선

손실 규모 기준의 위험 관리 지표 도입

취소율 중심 KPI → 손실 기반 KPI로 전환

8. 사용 기술

Python

Pandas, NumPy

Matplotlib / Seaborn

데이터 전처리 및 시각화

9. 프로젝트 의의

본 프로젝트는 데이터 분석을 통해
비즈니스 의사결정에 직접 연결되는 인사이트를 도출한 사례입니다.

단순 EDA를 넘어

실제 운영 리스크를 정량화하고

개선 방향까지 제안했다는 점에 의의가 있습니다.

10. 문제 발생 및 해결 과정
문제 1. 취소율과 실제 손실 규모가 일치하지 않는 문제

문제 상황
초기 분석에서 예약 유형별 취소율을 기준으로 위험도를 판단했으나,
취소율이 높은 예약이 반드시 큰 손실로 이어지지 않는 현상이 확인되었습니다.

해결 과정

취소 여부만 보던 관점에서 벗어나

취소 발생 시 손실 금액을 핵심 지표로 재정의

예약 조건 조합별 총 손실 / 평균 손실 지표를 추가 분석

해결 결과

취소율 중심 판단의 한계를 인식

실제 비즈니스 리스크를 반영한 위험 판단 가능

문제 2. 예약 조건 단일 변수 분석의 한계

문제 상황
환불 여부, 예약 채널, 보증금 조건을 각각 분리해 분석할 경우
위험 요인이 분산되어 명확한 기준을 도출하기 어려웠습니다.

해결 과정

주요 예약 조건을 조합한 복합 조건(조합) 단위 분석으로 전환

환불 여부 × 채널 × 보증금 유형 조합 생성

조합별 손실 규모 비교

해결 결과

특정 조건이 결합될 때 위험이 급격히 증가하는 구조 확인

“조건 자체가 아닌 조합이 위험을 만든다”는 인사이트 도출

문제 3. 분석 결과를 정책 제안으로 연결하기 어려운 문제

문제 상황
분석 결과는 도출되었으나,
운영 정책 관점에서 어떻게 활용할 수 있을지 명확하지 않았습니다.

해결 과정

고위험 조합을 기준으로

관리 우선 대상 예약 유형 정의

정책 변경 가능 지점(보증금, 채널 조건) 정리

단순 분석 결과가 아닌 개선 시나리오 형태로 재구성

해결 결과

데이터 분석 결과를 운영 정책 및 의사결정과 연결

실무 적용 가능한 분석 결과로 정리

11. 분석을 통해 얻은 교훈

지표 선택에 따라 분석의 방향과 결론이 완전히 달라질 수 있다

단일 변수 분석보다 비즈니스 맥락을 반영한 조합 분석이 중요하다

데이터 분석의 최종 목적은 설명이 아닌 의사결정 지원임을 재확인했다
